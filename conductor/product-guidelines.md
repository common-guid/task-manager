# Product Guidelines: Obsidian Task Manager

## Design Principles
- **Obsidian Native:** The UI must feel like a natural extension of Obsidian. Use standard CSS variables, UI components (modals, settings, ribbons), and iconography.
- **Minimalist Aesthetic:** Avoid clutter. Focus on the data (the tasks) and only show actions and metadata when necessary.
- **Consistency:** Ensure that layouts, fonts, and spacing are consistent across all views and settings.

## User Experience (UX)
- **Keyboard-First Design:** All primary actions should be accessible via hotkeys or the Command Palette. Support standard navigation shortcuts.
- **Instant Feedback:** Any action (e.g., checking a task, updating a priority) should provide immediate visual confirmation or state update.
- **Zero Configuration (Default):** The plugin should be useful immediately upon activation, with sensible defaults for Bases views and metadata.
- **Contextual Awareness:** Links between tasks and notes should be intuitive and easy to follow, maintaining the user's flow.

## Content & Tone
- **Professional & Precise:** Use clear, unambiguous labels for statuses, priorities, and UI elements.
- **Helpful & Instructive:** Provide useful tooltips and clear error messages to help users understand how to get the most out of the system.
- **Markdown-Centric:** All task data should be stored in a way that remains readable and editable in plain Markdown, adhering to the "local-first" philosophy of Obsidian.

## Branding
- **Subtle Identity:** The plugin should have a clear name ("Task Manager") but not compete with Obsidian's own branding. Use standard Obsidian iconography where possible.
- **Functional Naming:** Features should be named according to their function (e.g., "Bases Hub", "Dependency Map") for clarity.
